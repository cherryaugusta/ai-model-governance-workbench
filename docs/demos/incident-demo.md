# Incident Demo

The strongest operational demo path connects incident state to governed rollback and audit evidence.

## Demo sequence

1. Open the incidents dashboard.
2. Inspect an open high or critical incident.
3. Show its linked system or release context.
4. Navigate to the relevant release candidate.
5. Show rollback eligibility or prior rollback outcome.
6. Open the audit timeline.
7. Show the incident and rollback evidence trail.

## Why this demo matters

This scenario proves the platform is not only about release preparation. It also models what happens after an AI release causes operational concern.
